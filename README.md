# Facial Emotion Recognition ML Project

This project studies facial emotion recognition with a structured machine
learning workflow: dataset analysis, traditional ML baselines, CNN training,
and ML-enhanced CNN improvement.

The project is designed to avoid treating CNN as a black-box superior model.
Instead, it compares CNN with traditional ML methods and investigates how ML
features, classifiers, probability fusion, and error correction can improve or
explain CNN predictions.

## Run Order

Run scripts from the project root folder:

```bash
cd facial-emotion-recognition-ml-cnn
# or: cd path/to/project

python3 00_dataset_analysis.py
python3 01_traditional_ml_baselines.py
python3 02_cnn_baseline.py
python3 03_ml_enhanced_cnn.py
```

The scripts should be run in this order because later steps use the models or
results generated by earlier steps.

The current rerun uses the FER2013 download stored under `fer2013/`. The
`validation/` folder is used as the held-out evaluation split in the scripts.
Some output files keep the word `test` for backward compatibility, but in the
current rerun those files report metrics on `fer2013/validation`, the held-out
evaluation split.

## Step Overview

### 00 Dataset Analysis

Script:

```text
00_dataset_analysis.py
```

Purpose:

- Count train/validation images for each emotion class.
- Calculate class percentages and class imbalance ratio.
- Check unreadable or corrupted images.
- Generate class distribution charts.

Main outputs:

```text
results/dataset_analysis/dataset_analysis_summary.txt
results/dataset_analysis/split_class_distribution.csv
results/dataset_analysis/overall_class_distribution.csv
results/dataset_analysis/class_distribution_train_test.png
results/dataset_analysis/overall_class_percentage.png
```

### 01 Traditional ML Baselines

Script:

```text
01_traditional_ml_baselines.py
```

Purpose:

- Build non-deep-learning baselines for comparison.
- Evaluate whether handcrafted HOG features are better than raw pixels.
- Use validation Macro F1 for model selection because the dataset is class
  imbalanced.

Experiments:

- Raw pixels + SGD linear SVM
- HOG + Logistic Regression
- HOG + KNN
- HOG + SGD linear SVM

Main outputs:

```text
results/step1_result/test_metrics_summary.csv
results/step1_result/validation_tuning_results.csv
results/step1_result/selected_model_summary.csv
results/step1_result/classification_reports/
results/step1_result/figures/
```

Key interpretation:

HOG + KNN is the strongest traditional ML baseline, while raw pixels perform
much worse. This motivates using HOG and distance-based ML as complementary
signals in the later CNN-enhancement step.

### 02 CNN Baseline

Script:

```text
02_cnn_baseline.py
```

Purpose:

- Train CNN models for facial emotion recognition.
- Compare CNN training with and without CLAHE preprocessing.
- Use class-weighted loss to reduce the effect of class imbalance.
- Select the best checkpoint by validation Macro F1.

Experiments:

- CNN with CLAHE
- CNN without CLAHE

Main outputs:

```text
models/best_emotion_cnn_pytorch.pth
models/best_emotion_cnn_pytorch_with_clahe.pth
models/best_emotion_cnn_pytorch_without_clahe.pth
results/step2_result/experiment_comparison_summary.txt
results/step2_result/metrics/cnn_experiment_metrics_summary.csv
results/step2_result/metrics/training_history_*.csv
results/step2_result/classification_reports/
results/step2_result/figures/
```

Key interpretation:

The selected CNN becomes the baseline deep learning model for Step 03. It is
not treated as automatically superior; Step 03 tests whether traditional ML
methods can further improve or explain its predictions.

### 03 ML-Enhanced CNN

Script:

```text
03_ml_enhanced_cnn.py
```

Purpose:

- Load the best CNN from Step 02.
- Extract CNN deep features.
- Train traditional ML classifiers on CNN representations.
- Combine handcrafted HOG features with CNN features.
- Compare feature-level fusion, probability-level fusion, stacking, and
  confusion-aware correction.
- Generate Grad-CAM visual explanations.

Experiments:

- End-to-end CNN baseline
- CNN deep features + Logistic Regression / KNN / SGD linear SVM
- PCA-reduced CNN features + ML classifiers
- HOG + CNN feature fusion
- CNN probability + ML probability fusion
- Step1-guided confusion-aware ML correction
- Stacking over CNN and ML probability outputs
- Grad-CAM examples for correct and incorrect predictions

Main outputs:

```text
results/step3_result/objective3_summary.txt
results/step3_result/objective3_test_metrics.csv
results/step3_result/objective3_validation_metrics.csv
results/step3_result/confusion_correction/
results/step3_result/classification_reports/
results/step3_result/figures/
results/step3_result/gradcam/
```

Key interpretation:

Step 03 evaluates whether traditional ML can enhance CNN-based recognition.
The strongest hybrid model is selected using Macro F1, with attention to both
overall performance and minority-class behavior.

## Important Evaluation Metric

Macro F1 is emphasized because the dataset is class imbalanced. In the current
FER2013 split, Happy is the largest class and Disgust is the smallest class,
with an imbalance ratio of about 16.48:1. Accuracy can be dominated by frequent
classes such as Happy, while Macro F1 gives each emotion class equal weight.
This makes it more appropriate for comparing models on all seven emotions.

## Current Rerun Results

These results are from the rerun using `fer2013/train` and
`fer2013/validation`.

### Step 01 Traditional ML

| Method | Evaluation Accuracy | Evaluation Macro F1 | Evaluation Weighted F1 |
|---|---:|---:|---:|
| HOG + KNN | 0.4503 | 0.4391 | 0.4333 |
| HOG + Logistic Regression | 0.3862 | 0.3475 | 0.3694 |
| HOG + SGD Linear SVM | 0.3553 | 0.3019 | 0.3373 |
| Raw Pixel + SGD Linear SVM | 0.2828 | 0.2251 | 0.2690 |

### Step 02 CNN Baseline

| Model | Validation Macro F1 | Evaluation Accuracy | Evaluation Macro F1 | Evaluation Weighted F1 |
|---|---:|---:|---:|---:|
| CNN with CLAHE | 0.5878 | 0.5497 | 0.5248 | 0.5350 |
| CNN without CLAHE | 0.6123 | 0.5734 | 0.5571 | 0.5511 |

The selected CNN baseline is the CNN without CLAHE.

### Step 03 ML-Enhanced CNN

| Method | Evaluation Accuracy | Evaluation Macro F1 | Evaluation Weighted F1 |
|---|---:|---:|---:|
| CNN End-to-End | 0.5734 | 0.5571 | 0.5511 |
| CNN Deep Feature + Logistic Regression | 0.5804 | 0.5603 | 0.5616 |
| CNN Deep Feature Probability Fusion + Logistic Regression | 0.5821 | 0.5655 | 0.5619 |
| HOG-CNN Fusion Probability Fusion + KNN | 0.5809 | 0.5718 | 0.5607 |
| Stacking + Logistic Regression | 0.5879 | 0.5653 | 0.5686 |
| Stacking + Balanced Logistic Regression | 0.5865 | 0.5570 | 0.5670 |

The best evaluation Macro F1 model is **HOG-CNN Fusion Probability Fusion +
KNN**, improving Macro F1 from `0.5571` for the standalone CNN to `0.5718`.
